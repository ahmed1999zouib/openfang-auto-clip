# 版权转换指南

本文档说明 OpenFang Auto Clip 当前的三层内容转换思路，以及它们各自的适用范围和限制。

## 总览

直接下载并二次上传视频，常见风险包括：

- 版权投诉
- Content ID 匹配
- 平台收益归原作者
- 账号处罚或封禁

项目当前提供三层转换路径：

| 级别 | 速度 | 安全性 | 说明 |
|------|------|--------|------|
| Level 1 | 快 | 中等 | 视觉混音、速度调整、画面处理 |
| Level 2 | 慢 | 高 | 新脚本、新配音、保留核心信息 |
| Level 3 | 很慢 | 最高 | 全流程原创重制 |

## Level 1：视觉混音

适合：

- 评论型内容
- 教育类摘取
- 需要快速验证的短视频实验

特点：

- 基于 FFmpeg 的可复现处理
- 会改变画面、节奏和部分音频特征
- 当前是项目里最成熟、最容易落地的能力

限制：

- 不能把受保护角色自动变成安全资产
- 不适合把完整受版权保护内容直接重新发布

## Level 2：脚本重写

目标流程：

1. 转录原视频内容
2. 抽取核心信息
3. 重写脚本
4. 生成新配音
5. 配套替换素材

适合：

- 商业化账号
- 课程拆解
- 需要更高版权安全性的生产流程

当前状态：

- 已有设计方向和接口预留
- 仍属于脚手架能力，需要继续补齐

## Level 3：完全重制

目标流程：

1. 深度分析原内容结构
2. 重新编写原创脚本
3. 生成原创角色、镜头和场景
4. 生成原创配音和音乐
5. 完成独立成片

适合：

- 商业客户
- 白标内容生产
- 对原创性要求极高的场景

当前状态：

- 适合 roadmap 宣传
- 不应当在对外介绍中包装成已交付成品

## 法律与合规建议

- 美国、欧盟、中国、日本等地区的版权规则并不完全相同
- 项目提供的是“降低风险的技术路径”，不是法律意见
- 商业用途请务必结合本地法律环境评估

## 实操建议

### 最低可行路径

```bash
./auto_clip.sh "URL" --transform 1
```

### 更安全的对外定位

- 把 `Level 1` 说成视觉混音和低风险处理
- 把 `Level 2/3` 说成正在建设的高安全生产链路
- 如果素材依赖明显角色 IP，优先走 `Level 2` 或 `Level 3`

## 结论

`Level 1` 是当前最强的可用能力，但不是完整版权解决方案。项目对外叙事应该坚持真实、可验证、不过度承诺。
