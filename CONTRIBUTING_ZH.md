# 参与贡献指南

感谢你关注 OpenFang Auto Clip。欢迎通过 Issue、讨论和 Pull Request 参与项目。

## 如何开始

### 报告 Bug

提交前先搜索现有 Issue，避免重复。

建议在 Bug 报告中提供：

- 问题描述
- 复现步骤
- 期望结果
- 实际结果
- 运行环境
- 相关日志或截图

### 提出功能建议

建议说明：

- 你要解决的具体问题
- 期望的工作流或交互方式
- 已尝试过的替代方案
- 完成后的判断标准

## Pull Request 工作流

1. Fork 仓库并创建分支
2. 编写或修改代码
3. 补测试
4. 更新相关文档
5. 本地运行：
   - `python3 -m unittest discover -s tests`
6. 提交并发起 PR

## 代码标准

### Python

- 遵循 PEP 8
- 公开函数尽量补类型和文档字符串
- 新逻辑优先补测试

### Shell

- 使用 `set -euo pipefail`
- 为变量加引号
- 保持脚本可重复执行

## 当前最需要的贡献方向

- Level 1 效果优化与性能优化
- 更强的 benchmark 和 demo 资产
- Web 管理器可用性提升
- 文档同步维护
- 多平台导出和发布链路

## 交流方式

- Issues：缺陷和明确任务
- Discussions：方案讨论、路线图、问题交流
- PR：带验证结果的具体改动
